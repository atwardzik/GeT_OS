NAME

write - write output

SYNOPSIS

#include <libc.h>

int write(int file, void *buf, int len);

DESCRIPTION

write() attempts to write len bytes of data from from buffer pointed to by buf into object referenced by file descriptor. This function is based on the Virtual Filesystem (VFS) interface which encapsulates the driver internals while giving possibility to write specific data into it. Therefore it is possible to both write physical files as well as stdout or even pending data from an open socket.

PARAMETERS

  • file - descriptor of a file. Reserved file descriptors are:
    1. stdin
    2. stdout
    3. stderr
  • buf - buffer the data will be written into
  • len - length of the data that will be read. Please note it is not the length of the passed parameter buf. You must ensure the buffer is large enough to fit the data you are reading. Failing may cause undefined behaviour.

RETURN VALUES

Positive number corresponding to the number of bytes written into buf. When the data is unavailable due to some reasons that are expected by the driver, read may return 0. A negative error code is returned if an error occurs.

ERRORS

  • [EBADF] - file is not a valid file descriptor
  • [ENOMEM] - out of memory while trying to traverse the filesystem structure
